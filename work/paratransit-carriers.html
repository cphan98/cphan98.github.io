user-focused search: integrating paratransit carriers

ui/ux designer

|

3 days

this project focused on enhancing the usability of the operator offers page by integrating paratransit carriers and making their service type immediately scannable. by redesigning the list from a hidden accordion to a fully visible list with clarifying tags and improving the filtering system, i delivered significant user experience gains with minimal visual change to the existing page design.

the problem

the original operator offers page was incomplete, as it lacked job listing for paratransit carriers. this primary challenges were:

  1. missing information: the page did not include job offers for paratransit services.

  2. poor ux pattern: the complete list of existing carriers was hidden within an accordion, making it non-intuitive for users who wanted to quickly scan available options without interacting with a filter system.

  3. ambiguous roles: it was not immediately clear which carriers offered public transit services versus paratransit services.

figma - original operator offers page

goals

the main objectives were to:

  • integrate paratransit offers: add the missing paratransit carriers to the existing list.

  • enhance clarity via tags: clearly label carriers as either "paratransit" or "public transit" for instant recognition.

  • improve usability: display the full list of carriers by default, removing the problematic accordion element to improve discoverability.

  • streamline functionality: refine the filter system by removing the job application option and adding a "reset all filters" function.

my process

the process focused on maximizing ux improvement while minimizing visual redesign, as requested by the stakeholders.

identifying scannability solution

to address the need for clear sservice identification, i immediately proposed an designed carrier tags (e.g., "paratransit", "public transport") to be placed directly on each carrier listing.

example of a tag for a carrier listing

addressing ux friction

i challenged the existing design pattern of hiding the complete list in an accordion. i advocated for displaying the full list by default, arguing that full visibility (scannability) is a better user experience, especially for job seekers who are hunting for specific carriers.

streamlining filters

i followed the request to remove a a submission option from the filtering area. i also enhanced the filtering functionality by adding a "reset all filters" button for better user control.

filetring system before and after

final information architecture

despite the internal debate on carrier categorization (modes of transport vs. service types), i maintained the original categorization structure (categorized by modes of transport) to respect the constraint of not making major changes to the page's core information architecture.

figma - prototype of the final design

the solution

the final design provided clarity and improved usability for job seekers.

  • visible carrier list: the full list of carriers is now displayed by defaulting beneath the filtering system, eliminating the need for users to click and reveal content.

  • service tags: each carrier now features a clear tag (paratransit or public transport), allowing users to immediately identify the type of service offered.

  • enhanced filtering system:

    • new filter buttons were added for service type (paratransit/public transport).

    • a "reset all filters" button was implemented for quick filter management.

    • the job application option was removed to simplify the user's action flow.

a screenshot of exo's current operator offers page

reflections & learnings

the most challenging part of this project was navigating the constraint of making minor visual changes while delivering major improvements to the user experience. i learned that small additions, like clearly visibile tafs and removing unecessary friction (the accordion), can have a large impact on a user's ability to scan, filter, and successfully complete a goal. it reinforced the importance of challenging non-intuitive patterns, even when a simple modification is requested.
