rethinking paratransit onboarding with tutorial videos

ui/ux designer & integrator

|

10 days

the goal of this project was to integrate a seroes of tutorial videos into the paratransit section of the website to help users master online trip reservations. i successfully steered the project toward a better information architecture by creating three dedicated tutorial pages instead of overloading the main page, ensuring clarity, accessibility, and a fluid user flow.

the problem

while the existing paratransit page was functional, the team recognized an opportunity to improve user self-service. the challenge was to integrate three categories of tutorial content--covering account creation, trip booking, and trip management--onto the page without overwhelming the user or compromising accessibility. the three video categories were:

  • create and use customer account for paratransit

  • book paratransit for occasional travel

  • view, modify, or cancel a trip

figma - prototype of the original paratransit page

goals

the main goals for this project were to:

  • improve clarity: clearly communicate the steps for making an online paratransit reservation.

  • guide users: provide detailed, step-by-step guidance through new tutorial videos.

  • optimize information architecture: design a structure that accommodates the new video content without creating a single, overly long, or cluttered page.

  • enhance usability: ensure the final pages are simple, accessible, and promote fluid user navigation.

my process

my process involved strategic design recommendations to optimize the user experience.

initial design

i initially designed a full page redesign to accommodate the videos. although the team only wanted to add new content, i presented two layout options for the videos: (a) all videos on the main page or (b) separate pages for each of the three categories.

strategic recommendation

i successfully recommended option b (separate pages) to avoid overwhelming the user and to keep the main paratransit page focused and clear. the team accepted this recommendation.

enhancing navigation

when asked how the three new pages would be connected, i proposed adding linked cards in the footer for each page, providing a clear and coherent pathway between the three tutorial pages, which was vital for a fluid user journey.

examples of cross-navigation cards

final page design

all tutorial pages follow a consistent, accessible strucutre: an introduction, a table of contents with hyperlinks, sections with text and embedded youtube videos, and footer cards for cross-navigation.

main page modifications

i also updated the main paratransit page by:

  • transforming the "three easy steps" section into distinct, easy-to-read cards.

  • adding a clear call-to-action section: "what to know before booking online."

  • adding a clear call-to-action section for the "need help getting started?" section linking to the three new tutorial pages.

figma - prototype of the modified paratransit page

the solution

the solution was a set of three user-centric tutorial pages and a cleaner main paratransit page:

  • simplified main page: the main page now directs users to specific tutorials via a clean section with three distinct links, preventing content overload.

  • focused tutorial pages: by separating the content, each of the three new pages is highly focused on a signle task (e.g., book paratransit), ensuring users are not distracted.

  • improved scannability: the simple, accessible design, coupled with an internal table of contents, allows users to quickly jump to the exact step they need.

  • fluid navigation: the linked footer cards ensure users can easily navigate between the three related tutorial pages without returning to the main page.

reflections & learnings

the main lesson learned was the critical role of communication and proactive recommendation. the initial brief lacked a clear structural vision for the new content. by proactively presenting the separate-page solution and, later, anticipating the need for cross-page navigation, i not only fulfilled the requirements but optimized the user experience. this underscored that continuous communication is necessary to align stakeholder requirements with optimal user experience design.
