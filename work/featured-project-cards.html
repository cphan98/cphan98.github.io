enhancing featured project cards for improved scannability

ui/ux designer & html integrator

|

1 week

faced with an upcoming increased in the number of projects, i redesigned exo's project cards to improve scannability and clarity on the "projects and achievements" page. by implementing category tags and putting emphisis on ongoing projects, i ensured users can quickly understand the subject matter of each project, setting the foundation for future organizational needs.

the problem

with an increased number of pilot projects and achievements anticipated, the existing project card format on exo's page lacked sufficient detail to effectively communicate the project's subject matter at a glance. the need was to:

  • increased clarity on what each project is about.

  • establish an organizational structure (like categorization) to handler future growth in project listings.

my process

research & competitive analysis

after the project briefing, i researched similar organizations, using artm's projects page as a key reference point to explore different categorization strategies.

artm's projects page

ideation & prototyping

i developed two main design options:

  • option a (categorized pages): projects were separated into different category pages.

  • option b (tagged landing page): all projects remained on the landing page but included clear category tags.

option a - figma prototype of exo's page "projects and achievements" with category tags

option b - figma prototype of exo's page "projects and achievements" without category tags

stakeholder review

i presented both options to the content strategy team. we concluded that option a (separate category pages) would create too many clicks and risked losing the user due to too many steps.

final design decision

given the current number of projects, we decided to proceed with option b (tagged landing page), implementing category tags only on the ongoing project cards.

option b (modified) - figma prototype of exo's page "projects and achievements" with category tags only in the ongoing projects cards

goals

the main goals for this enhancement were to:

  • increased scannability: allow users to quickly identify the theme of a project without reading the full description.

  • improve clarity: immediately communicate the core sibject matter using visual indicators (tags).

  • establish future organization: lay the groundwork for categorization, enabling the site to scale cleanly as more projects are added.

  • reduce cognitive load: make it easier for users to filter and process the information on crowded page.

the solution

the solution centered on introducing clear and visually distinct metadata to the project cards:

  • category tags: prominent, easily scannable category tags (e.g., "infrastructure works", "technological innovation", "fleet and maintenance") were added directly above the project title on the ongoing project cards.

  • visual priority: the tags were designed in a high-contrast black color to quickly draw the user's attention, which makes them easily scannable.

  • improved scannability: when a user lands on the page, the ongoing projects section is immediately visible, and the tags provide instant contact on the project's nature, improving the first-sight understanding.

key deliverables

  • finalized ui/ux design in figma for the enhanced project cards.

  • implemented html and css code for the new, tagged project card component.

next steps

while the immediate solution addressed clarity, the long-term design anticipates the increased volume of future projects. the planned future page design incorporate full categorization to maintain scannability:

  • future design: projects will be grouped under clear section headers.

  • continued clarity: the category tags will remain on the ongoing projects cards. this design ensures that as the page grows, the user will still have a clear, organized view and can easily jump to the category of interest.

figma - prototype of a seggested future design of exo's "projects and achievements" page

reflections & learnings

my main takeaway was that user effort and experience often outweigh pure organizational logic. although separating projects into different category pages seemed more organized, it introduced too much friction (clicks) for the user. it is crucial to consider the user's journey when consulting the projects and achievements page, balancing structure with minimizing effort.
